# Admin Portal Screen Toggles — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let SUPER_ADMIN toggle which dashboard screens are visible to ADMIN users on the admin portal (`admin.carferry.online`), controlling what can be edited in the copy database.

**Architecture:** A new `admin_screen_toggles` database table stores per-screen enabled/disabled state. The `/api/auth/me` endpoint filters `menu_items` against these toggles when `ADMIN_PORTAL_MODE=true`. SUPER_ADMIN always sees all screens + the toggle management UI. ADMIN users only see enabled screens. A new "Screen Access" tab under System Settings provides the toggle UI.

**Tech Stack:** FastAPI, SQLAlchemy 2.0 async, Alembic, React 19, Next.js 16, Tailwind CSS v4, shadcn/ui

    def __repr__(self) -> str:

## Task 2: Create the service layer

**Files:**
- Create: `backend/app/services/admin_screen_service.py`

The service handles listing all toggles (seeding defaults if the table is empty) and bulk-updating them.

- [ ] **Step 1: Create the service**

```python
# backend/app/services/admin_screen_service.py
from sqlalchemy import select
from sqlalchemy.ext.asyncio import AsyncSession

from app.models.admin_screen_toggle import AdminScreenToggle

# Screens that can be toggled — Dashboard and System Settings are always on
TOGGLEABLE_SCREENS: list[str] = [
    "Users",
    "Ferries",
    "Branches",
    "Routes",
    "Schedules",
    "Items",
    "Item Rates",
    "Payment Modes",
    "Ticketing",
    "Multi-Ticketing",
    "Reports",
    "Rate Change Logs",
    "Employee Transfer",
    "Ticket Verification",
    "User Sessions",
]

# These screens are always visible and cannot be turned off
ALWAYS_ON_SCREENS: set[str] = {"Dashboard", "System Settings"}


async def _seed_defaults(db: AsyncSession) -> None:
    """Insert default rows for any missing toggleable screens."""
    result = await db.execute(select(AdminScreenToggle.screen_name))
    existing = {row[0] for row in result.all()}
    for name in TOGGLEABLE_SCREENS:
        if name not in existing:
            db.add(AdminScreenToggle(screen_name=name, is_enabled=True))
    if TOGGLEABLE_SCREENS and set(TOGGLEABLE_SCREENS) - existing:
        await db.flush()


async def get_all_toggles(db: AsyncSession) -> list[AdminScreenToggle]:
    """Return all toggles, seeding defaults for any missing screens."""
    await _seed_defaults(db)
    result = await db.execute(
        select(AdminScreenToggle).order_by(AdminScreenToggle.id)
    )
    return list(result.scalars().all())


async def get_enabled_screens(db: AsyncSession) -> set[str]:
    """Return the set of screen names that are currently enabled."""
    await _seed_defaults(db)
    result = await db.execute(
        select(AdminScreenToggle.screen_name).where(
            AdminScreenToggle.is_enabled == True
        )
    )
    enabled = {row[0] for row in result.all()}
    return enabled | ALWAYS_ON_SCREENS


async def bulk_update_toggles(
    db: AsyncSession, updates: dict[str, bool]
) -> list[AdminScreenToggle]:
    """Update multiple screen toggles at once. Returns all toggles after update."""
    await _seed_defaults(db)
    result = await db.execute(select(AdminScreenToggle))
    all_toggles = {t.screen_name: t for t in result.scalars().all()}
    for screen_name, is_enabled in updates.items():
        if screen_name in all_toggles and screen_name not in ALWAYS_ON_SCREENS:
            all_toggles[screen_name].is_enabled = is_enabled
    await db.flush()
    return sorted(all_toggles.values(), key=lambda t: t.id)
```

async def me(current_user: User = Depends(get_current_user), db: AsyncSession = Depends(get_db)):

## Task 5: Frontend — Screen Access tab component

**Files:**
- Create: `frontend/src/app/dashboard/settings/components/screen-access-tab.tsx`

A card with toggle switches for each screen. Fetches current state on mount, saves on each toggle change.

- [ ] **Step 1: Create the component**

```tsx
// frontend/src/app/dashboard/settings/components/screen-access-tab.tsx
"use client";

import { useEffect, useState } from "react";
import api from "@/lib/api";
import { Card, CardContent, CardHeader, CardTitle } from "@/components/ui/card";
import { Switch } from "@/components/ui/switch";
import { Label } from "@/components/ui/label";
import { Skeleton } from "@/components/ui/skeleton";
import { ToggleLeft } from "lucide-react";

interface ScreenToggle {
  id: number;
  screen_name: string;
  is_enabled: boolean;
}

export default function ScreenAccessTab() {
  const [toggles, setToggles] = useState<ScreenToggle[]>([]);
  const [loading, setLoading] = useState(true);
  const [saving, setSaving] = useState<string | null>(null);
  const [error, setError] = useState("");

  useEffect(() => {
    (async () => {
      try {
        const { data } = await api.get<ScreenToggle[]>("/api/settings/screen-toggles");
        setToggles(data);
      } catch {
        setError("Failed to load screen toggles.");
      } finally {
        setLoading(false);
      }
    })();
  }, []);

  const handleToggle = async (screenName: string, checked: boolean) => {
    setError("");
    setSaving(screenName);
    try {
      const { data } = await api.put<ScreenToggle[]>("/api/settings/screen-toggles", {
        toggles: { [screenName]: checked },
      });
      setToggles(data);
    } catch (err: unknown) {
      const msg =
        (err as { response?: { data?: { detail?: string } } })?.response?.data
          ?.detail || "Failed to update toggle.";
      setError(msg);
    } finally {
      setSaving(null);
    }
  };

  if (loading) {
    return (
      <div className="space-y-3">
        {[1, 2, 3, 4, 5].map((i) => (
          <Skeleton key={i} className="h-16 w-full rounded-lg" />
        ))}
      </div>
    );
  }

  return (
    <Card>
      <CardHeader>
        <CardTitle className="flex items-center gap-2 text-lg">
          <ToggleLeft className="h-5 w-5" />
          Admin Portal Screen Access
        </CardTitle>
      </CardHeader>
      <CardContent className="space-y-4">
        <p className="text-xs text-muted-foreground leading-relaxed">
          Control which screens are visible to Admin users on the admin portal.
          Disabled screens will be completely hidden from the sidebar.
          Dashboard and System Settings are always visible.
        </p>

        <div className="space-y-3">
          {toggles.map((toggle) => (
            <div
              key={toggle.id}
              className="flex items-center justify-between rounded-lg border p-4"
            >
              <div className="space-y-0.5">
                <Label
                  htmlFor={`screen-${toggle.id}`}
                  className="text-sm font-medium"
                >
                  {toggle.screen_name}
                </Label>
              </div>
              <Switch
                id={`screen-${toggle.id}`}
                checked={toggle.is_enabled}
                onCheckedChange={(checked) =>
                  handleToggle(toggle.screen_name, checked)
                }
                disabled={saving === toggle.screen_name}
              />
            </div>
          ))}
        </div>

        {error && (
          <p className="text-sm text-destructive bg-destructive/10 border border-destructive/20 rounded p-2">
            {error}
          </p>
        )}
      </CardContent>
    </Card>
  );
}
```

## Design Decisions

1. **Database over config file**: Toggles persist across container rebuilds and `refresh_admin_db.sh` runs (the refresh script restores `ssmspl_sync` → `ssmspl_admin`, but `admin_screen_toggles` is admin-portal-only data that doesn't exist in `ssmspl_sync`, so it's preserved by `--clean --if-exists` since there's no matching table to drop/recreate).

2. **Auto-seeding**: The service seeds default rows on first access rather than requiring a data migration. This handles both fresh deployments and adding new screens later.

3. **SUPER_ADMIN always sees everything**: Toggles only filter ADMIN users' menus. SUPER_ADMIN sees all screens regardless, ensuring they can never lock themselves out.

4. **Server-side filtering**: Menu items are filtered in `/api/auth/me` before reaching the frontend. The sidebar already uses `menu_items` to determine visibility, so no frontend sidebar changes needed.

5. **`refresh_admin_db.sh` safety**: The `admin_screen_toggles` table exists only in `ssmspl_admin` (created by migration), not in `ssmspl_sync`. The `--clean --if-exists` flags in the refresh script generate `DROP TABLE IF EXISTS` for tables in the dump — since `admin_screen_toggles` is never in the dump (it's not in `ssmspl_sync`), it won't be dropped. Toggle state survives DB refreshes.
